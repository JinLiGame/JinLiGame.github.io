知识点总结

发表于 2019-11-19 更新于 2020-03-13 分类于 Unity 阅读次数：
对最近的内容进行总结、整理和归纳。

DrawCall 合批

  • DrawCall计算方式：计算Depth并升序排列，相同Depth的按Material的InstanceID升序排列，相同Depth和Material的按Texture的InstanceID升序排列。
  • 优化方式：使用图集或者有时候为了合并层级，比如Text，就需要给Text垫高层级，即将Text下面放一个其他图集中的透明图片，从而使得层级整齐，以达到该Text与其他Text层级相同而能合批的目的。
  • 参考：UGUI DrawCall

避免网格重建

  • 优化方式：将一直不变化的UI元素放在一个 Canvas 中；一直变化的如跑马灯，或者一直在做动画的UI元素放在另一个 Canvas 中。

  • 参考：UGUI Rebuild

减少 OverDraw

  • 优化方式：
    1. 在ImageType选项为Sliced的情况下，不需要Fill Center 的时候去掉勾选
    2. 避免短时间内产生大量特效（不合理的多层特效，不仅效率低下，而且那么多层特效能看见的只能前面的几个，做多了也没用）
    3. 让特效在屏幕上均匀一些（比如：一个特效销毁后再产生另一个特效，轮流显示，这样总体压力减少，效果却不减）
    4. 避免使用内置的OutLine，Shadow组件，使用其他方案代替，比如：TextMeshPro。
  • 参考：
    1. UI优化技巧进阶：粒子、Raycast Target和OverDraw
    2. Unity 优化之UGUI (2017年版【一】)

减少Raycast Target

  • 将不需要点击事件的UI元素，比如：Text、Image和RawImage中的Raycast Target取消勾选。
  • 可以用 RaycastTarget检测小工具 ，在对应的editor模式下，把开启Raycast Target选项的UI以蓝色线框的形式显示出来，方便大家检查遗漏的关闭的UI元素
